# ESP32-S3 音频回声/自检工程

功能：开机自检播放 Beep -> 进入 Mic 监听模式 -> 实时将声音通过喇叭播放。

## 烧录
1. **拔掉 USB**。
2. **按住 BOOT 键，插入 USB**。
3. **松开 BOOT 键**。
4. 运行 `一键烧录.bat` 或 `pio run -t upload`。
5. **按一下 RST 键** 复位运行。

## 验证步骤 (一定要按顺序)

### 步骤 1: 听声 (Beep)
复位后，板子会等待 1 秒，然后播放 **300ms 的 "嘀" 声** (1kHz 正弦波)。
- **如果听到了**：说明 ESP32 -> MAX98357A -> 喇叭 链路完全正常！🎉
- **没听到**：
  - 检查 **SD** 脚是否接了 3.3V (重要!)
  - 检查喇叭有没有坏，线有没有断。
  - 确保接的是 **SPK+ / SPK-** (不要接地)。

### 步骤 2: 看灯/串口
- 打开串口监视器 (115200)。
- 应该能看到 `Mic Level: xxxx` 在跳动。
- 对着麦克风说话，板子上的 **RGB 灯** 会闪红光。
- 这说明麦克风采集正常。

### 步骤 3: 测回声 (Echo)
- 对着麦克风说话，喇叭应该能同步播出你的声音。
- **啸叫警告**：如果听到刺耳尖叫，请把麦克风和喇叭拿远一点，代码会自动降低增益来保护，但物理隔离最有效。

## 常见故障

| 现象 | 原因 | 解决 |
| :--- | :--- | :--- |
| **Mic Level 一直 0** | I2S RX 时钟或数据线没通 | 检查 SCK(5)/WS(4)/SD(6) 接线 |
| **Beep 不响** | I2S TX 或功放静音 | 检查 SD 脚是否接高电平 / 检查电源 |
| **全是噪音/电流声** | 地线干扰或增益过大 | 检查 GND 是否共地良好 |
| **声音断断续续** | 烧录或复位问题 | 按 RST 重启，或检查 5V 供电是否足 |
