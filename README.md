# Smart Baby Bed (智能婴儿床项目)

这是一个基于物联网（IoT）和人工智能（AI）技术的全栈智能婴儿床开源项目。项目涵盖了底层硬件控制（STM32）、音视频流媒体传输（ESP32 & OrangePi）、AI 姿态估计边缘计算以及移动端 Android App开发。

## 🌟 核心功能特性

* **智能环境调节**：通过 DS18B20 温度传感器和底板湿度传感器，自动判断婴儿是否发烧或尿床，并自动控制继电器开闭风扇、加热垫和蜂鸣器。
* **智能安抚系统**：声音传感器（麦克风）实时监测婴儿哭声，检测到哭泣时自动播放安抚音乐，并驱动步进电机实现摇篮自动摇晃功能。
* **AI 婴儿安全守护**：Orange Pi 搭载 RKNN 神经网络加速计算跑 YOLOv8-pose 模型，实时进行宝宝睡姿和面部遮挡检测（如危险的趴睡、口鼻遮挡），一旦发现危险立刻通过 MQTT 发送警报。
* **低延迟双向语音**：ESP32 结合 INMP441 麦克风和 MAX98357A 扬声器，利用 UDP 协议和 Android App 实现局域网内低延迟的双向对讲。
* **全景状态监控**：Android App 可实时查阅各种传感器状态，观看 MJPEG 视频流，并可对风扇、加热垫、音乐、摇篮进行强制手动控制。

## 📁 仓库代码结构 / Directory Structure

- `stm32/`：STM32F103 下位机源码（裸机超级循环与定时器中断架构版本）。
- `stm32freertos/`：STM32F103 下位机源码（全面升级为 FreeRTOS 实时操作系统架构版本，附带高并发多任务处理机制）。
- `OrangePi/`：运行在香橙派上的 Python 边缘计算端，包含 AI 姿态检测模型和 MJPEG Flask 视频流分发服务器。
- `esp32_gimbal_foc/`：运行在 ESP32 上的双向 UDP 音频对讲固件（基于 Arduino 框架）。
- `Baby/BabyBedApp/`：Android 端应用源码，使用 Java 开发，集成了 Paho MQTT 客户端和 UDP 音频直连模块。

## 🛠️ 技术栈与硬件选型

### 下位机与机电执行端
* **MCU**：STM32F103C8T6
* **OS**: Bare-metal Super Loop / FreeRTOS v11.1
* **传感器**：DS18B20（单总线温度）、声音传感器模块（GPIO 哭声识别）、ADC薄膜传感器（模拟量尿床检测）
* **执行器**：继电器模块控制（大电机风扇、加热片）、有源蜂鸣器、步进电机模块与ULN2003（摇床开环控制）
* **显示与通信**：0.96 寸 OLED 屏 (I2C)、ESP8266 Wi-Fi 芯片 (UART AT指令全功能控制)

### 边缘AI与中枢监控端
* **主控**：Orange Pi (香橙派)
* **AI 框架**：RKNN-Lite (NPU 硬件单元加速) + YOLOv8-pose 人体关键点检测模型
* **网络通讯件**：利用局域网 EMQX 部署私有 MQTT Broker；集成 Tailscale 构建虚拟局域网（VPN），实现跨公网与内网设备的无缝双向访问。

### 语音点对点对讲端
* **MCU**: ESP32
* **音频外设**: INMP441（I2S 数字麦克风）、MAX98357A（I2S 功放解码扬声器模块）

## 🚀 系统核心工作流运行图

1. **底层感知与动作决策**：STM32 利用多任务系统周期性采集温度和湿度数据，当阈值超标时立刻进行继电器级物理干预动作判定（高低电平触发）。
2. **高算力视觉安防**：OrangePi USB摄像头不断捕获床内画面。NPU 对每帧画面推理婴儿骨骼关键点（17 个点），结合自定义的防错判“平滑迟滞阈值滤波算法”，若系统研判婴儿长期处于面朝下的趴睡危城状态，立即向本地 EMQX Broker 上报 MQTT JSON格式红色告警。
3. **万物互联控制中枢**：STM32 通过串口透传，让 ESP8266 维持远端 MQTT 会话。所有的控制（单片机、香橙派、家长手机 App）均通过此私有部署的 MQTT Topic 异步串联。
4. **低延迟无感并发流**：视频与高频双向音频不走耗时、昂贵的公有云转收发（极力降低延时）。直接在设备之间通过 HTTP MJPEG（局域网摄像头）和 UDP Socket（语音打洞）建立点对点 P2P 直连。
