**A.1 边缘AI姿态分析核心逻辑 (Python/RKNN)**

```python
# babycam_unified.py - Posture Analysis Logic

def analyze_pose(self, kpts):
    """
    基于关键点几何规则的婴儿姿态与危险检测算法
    """
    if kpts is None or len(kpts) < 11: 
        return False, "No Person", "Unknown"
    
    # 获取关键点置信度
    nose = kpts[0]; nose_conf = nose[2]
    lear = kpts[3]; lear_conf = lear[2]
    rear = kpts[4]; rear_conf = rear[2]
    
    danger = False
    pose_text = "正常"
    
    # 1. 趴睡检测 (Prone Detection)
    # 逻辑：处于俯视视角时，若双耳清晰可见(置信度高)但面部五官模糊(置信度低)，判定为趴睡
    avg_face_conf = (nose[2] + kpts[1][2] + kpts[2][2]) / 3.0
    avg_ear_conf = (lear_conf + rear_conf) / 2.0
    
    if avg_ear_conf > 0.5 and avg_face_conf < 0.35:
        danger = True
        pose_text = "危险: 趴睡 (Face Down)"
    
    # 2. 遮挡检测 (Covering Detection)
    # 逻辑：构建以鼻子为中心的椭圆区域，若手腕关键点长时间处于该区域，判定为遮挡口鼻
    eye_dist = np.linalg.norm(kpts[1][:2] - kpts[2][:2]) # 双眼间距作为参考尺度
    RAD_X = eye_dist * 5.0
    RAD_Y = eye_dist * 3.2
    
    if self.is_in_danger_zone(kpts[9], nose, RAD_X, RAD_Y) or \
       self.is_in_danger_zone(kpts[10], nose, RAD_X, RAD_Y):
        danger = True
        pose_text = "危险: 口鼻遮挡"
        
    return danger, pose_text

```

// MqttManager.java - Core Communication

public void publishCmd(String key, int value) {
    try {
        JSONObject json = new JSONObject();
        json.put("cmd_id", System.currentTimeMillis());
        json.put("key", key);
        json.put("val", value);
        
        MqttMessage message = new MqttMessage(json.toString().getBytes());
        message.setQos(1); // QoS 1 确保控制指令至少送达一次
        client.publish("device/esp8266_01/cmd", message);
    } catch (Exception e) {
        Log.e(TAG, "Publish failed", e);
    }
}

## 附录B 硬件原理图说明

由于篇幅限制，此处对核心硬件电路连接进行文字描述：

1.  **STM32最小系统**：
    *   OSC_IN/OUT: 接8MHz晶振及22pF起振电容。
    *   NRST: 接10k上拉电阻及0.1uF电容至地（复位电路）。
    *   Boot0/1: 接地（从Flash启动）。

2.  **传感器接口**：
    *   **DS18B20**: VCC(3.3V), GND, DQ接STM32-PB12 (需4.7k上拉)。
    *   **声音模块**: VCC, GND, DO接STM32-PA6 (数字输入)。
    *   **湿度模块**: VCC, GND, AO接STM32-PA4 (ADC输入)。

3.  **执行器驱动**：
    *   **继电器**: S8550三极管驱动，基极接STM32-PA0/PA1，集电极接继电器线圈，发射极接5V。
    *   **L9110S电机驱动**: IA接PB8, IB接PB9，控制摇床电机正反转。
